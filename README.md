# Проект платформы для онлайн-обучения
1. Проект создан в рамках обучения по темам Django REST Framework и Docker. 
2. Реализована серверная часть приложения платформы для онлайн-обучения, на которой каждый желающий сможет размещать 
свои полезные материалы или курсы. Бэкенд-сервер возвращает клиенту ответы в виде JSON-структуры.
3. В проекте реализованы принципы CI/CD с использованием платформы GitHub Actions и Docker, в результате выполняется
автоматизированный деплой приложения на виртуальную машину, созданную на Yandex Cloud.

4. Для работы с приложением выполнить следующие инструкции по настройке удаленного сервера и деплоя:
А. Например, на Yandex Cloud создать виртуальную машину (ВМ): https://yandex.cloud/ru. Для этого нужно 
зарегистрироваться, создать платежный аккаунт, в панели управления Yandex Cloud выбрать раздел "Compute Cloud", 
нажать на кнопку «Создать ресурс» и выбрать в списке «Виртуальная машина». Выбрать операционную систему для виртуальной
машины (Ubuntu) и настроить конфигурацию машины.
Б. Настроить доступ к ВМ через SSH-ключ. Для этого в разделе «Доступ» указать логин (напр. test) и публичную часть 
SSH-ключа.
В. В терминале открыть командную строку и ввести команду для подключения: ssh ваш_пользователь@IP_адрес_машины.
Здесь ваш_пользователь заменить на логин из п.4Б, IP_адрес_машины - на «Внутренний IPv4» адрес ВМ (см. свойства своей
ВМ). Пример: ssh test@158.160.203.200. При необходимости ввести пароль от SSH-ключа.
Г. Выполнить обновление системы: sudo apt update.
Д. Выполнить команду для обновления установленных пакетов до их последних версий: sudo apt upgrade и подтвердить (y).
Е. Установить Docker согласно инструкции на официальном сайте: https://docs.docker.com/engine/install/ubuntu/#install-using-the-repository.
Для этого поочередно выполнить команды из инструкции.
Установить последнюю версию Docker: sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin.
Чтобы убедиться, что Docker запущен и правильно работает, выполнить команду: sudo docker run hello-world.
Ж. Настроить файрвол. Для этого проверить его состояние с помощью команды: sudo ufw status. 
Если файрвол отключен (Status: inactive), активировать его: sudo ufw enable и подтвердить (y).
Открыть необходимые порты для HTTP, HTTPS и обеспечения доступа к серверу по SSH (80, 443, 22):
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 22/tcp
Проверить настройки файрвола, чтобы убедиться, что правила применились: sudo ufw status
В результате выполнения команды должны увидеть, что порты 22, 80 и 443 находятся в состоянии ALLOW.

5. Клонировать репозиторий с проектом по ссылке: https://github.com/SergejGorshkov/DRFProject.git.
6. Для деплоя приложения настроить следующие переменные окружения в Secrets GitHub: DATABASE_HOST, DATABASE_NAME, 
DATABASE_PASSWORD, DATABASE_PORT, DATABASE_USER, DEPLOY_DIR, DOCKER_HUB_ACCESS_TOKEN, DOCKER_HUB_USERNAME, SECRET_KEY,
SERVER_IP, SSH_KEY, SSH_PASSPHRASE (при необходимости), SSH_USER.
7. Разворачивание приложения на сервере происходит при каждом push в репозиторий.
8. Приложение будет доступно по адресу: http://IP_адрес_машины (см п. 4В).
